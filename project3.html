Description du Projet

Ce projet implémente un réseau de neurones convolutif (CNN) pour classifier des images en trois catégories : chats, chiens et pandas. L'application web développée avec Flask et Gradio permet aux utilisateurs d'uploader une image et d'obtenir la prédiction du modèle en temps réel.

Dataset Utilisé

Source Nombre d'images Répartition Dimensions Augmentation
Kaggle + Images personnelles 8,500 images Chats: 3,000 • Chiens: 3,000 • Pandas: 2,500 224x224 pixels Rotation, Zoom, Flip, Brightness

Architecture du Modèle CNN

# Architecture du modèle CNN avec TensorFlow/Keras
model = Sequential([
    # Couche de convolution 1
    Conv2D(32, (3, 3), activation='relu', input_shape=(224, 224, 3)),
    MaxPooling2D(2, 2),
    Dropout(0.25),
    
    # Couche de convolution 2
    Conv2D(64, (3, 3), activation='relu'),
    MaxPooling2D(2, 2),
    Dropout(0.25),
    
    # Couche de convolution 3
    Conv2D(128, (3, 3), activation='relu'),
    MaxPooling2D(2, 2),
    Dropout(0.25),
    
    # Couches fully connected
    Flatten(),
    Dense(512, activation='relu'),
    Dropout(0.5),
    Dense(3, activation='softmax')  # 3 classes: cats, dogs, pandas
])

# Compilation
model.compile(
    optimizer=Adam(learning_rate=0.0001),
    loss='categorical_crossentropy',
    metrics=['accuracy']
])

Interface Gradio

L'interface utilisateur a été développée avec Gradio, une bibliothèque Python permettant de créer rapidement des interfaces web pour les modèles ML :

import gradio as gr
import tensorflow as tf
from PIL import Image
import numpy as np

# Chargement du modèle
model = tf.keras.models.load_model('cnn_model.h5')

def classify_image(image):
    # Prétraitement
    img = Image.fromarray(image.astype('uint8'), 'RGB')
    img = img.resize((224, 224))
    img_array = np.array(img) / 255.0
    img_array = np.expand_dims(img_array, axis=0)
    
    # Prédiction
    predictions = model.predict(img_array)
    class_names = ['Cat', 'Dog', 'Panda']
    
    # Résultats formatés
    results = {
        class_names[i]: float(predictions[0][i])
        for i in range(len(class_names))
    }
    
    return results

# Création de l'interface
interface = gr.Interface(
    fn=classify_image,
    inputs=gr.Image(type="numpy"),
    outputs=gr.Label(num_top_classes=3),
    title="Cats vs Dogs vs Pandas Classifier",
    description="Upload an image of a cat, dog, or panda to get classification results"
)

interface.launch()

Processus d'Entraînement

Étape 1 : Collecte et préparation des données

Téléchargement des datasets Kaggle, nettoyage des images, redimensionnement à 224x224px, et séparation train/validation/test (70%/15%/15%).

Étape 2 : Augmentation des données

Application de transformations pour améliorer la généralisation : rotation (±30°), zoom (0.8-1.2), flip horizontal, ajustement de luminosité.

Étape 3 : Architecture et entraînement

Développement d'une architecture CNN personnalisée avec 3 couches convolutives, dropout pour éviter l'overfitting, et early stopping.

Étape 4 : Évaluation et optimisation

Analyse des performances via matrice de confusion, courbes ROC, et fine-tuning des hyperparamètres.

Étape 5 : Développement de l'interface

Création d'une interface utilisateur intuitive avec Gradio permettant l'upload d'images et l'affichage des prédictions avec scores de confiance.

Étape 6 : Déploiement

Déploiement sur Hugging Face Spaces avec configuration des ressources GPU pour l'inférence.

Challenges et Solutions

Challenge 1 : Déséquilibre des classes

Solution : Utilisation de class weights pendant l'entraînement et oversampling de la classe minoritaire (pandas).

Challenge 2 : Overfitting

Solution : Implémentation de dropout layers, data augmentation poussée, et early stopping.

Challenge 3 : Performance en temps réel

Solution : Optimisation du modèle avec TensorFlow Lite et cache des prédictions.

Challenge 4 : Déploiement facile

Solution : Choix de Hugging Face Spaces pour un déploiement gratuit et simple avec support GPU.
