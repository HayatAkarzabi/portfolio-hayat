Contexte du Projet

Dans le cadre de mon stage PFA, j'ai développé un tableau de bord Power BI pour analyser les données du système webman_prod de traitement d'actes d'état civil. L'objectif était de centraliser et visualiser les données de 15 provinces, 220 bureaux, et plus de 3 millions d'actes traités.

Processus ETL Implémenté

Extract - Transform - Load

1. Extraction

  • PostgreSQL: Base principale webman_prod
  • MySQL: Données historiques
  • API REST: Données temps réel
  • Fichiers CSV/JSON: Exports manuels

2. Transformation

  • Nettoyage des données
  • Normalisation des formats
  • Création de mesures DAX
  • Établissement des relations
  • Gestion des valeurs manquantes

3. Chargement

  • Modèle de données Power BI
  • Actualisation automatique
  • Sécurité par niveau (RLS)
  • Optimisation des performances

Mesures DAX Clés

Indicateurs de Performance

// Taux de rejet par bureau
Taux_Rejet = 
DIVIDE(
    COUNTROWS(FILTER(Rejets, Rejets[statut] = "Rejeté")),
    COUNTROWS(Rejets),
    0
)

// Nombre d'actes traités aujourd'hui
ActesAujourdhui = 
CALCULATE(
    COUNTROWS(Actes),
    Actes[date] = TODAY()
)

// Performance comparative bureaux
PerformanceRelative = 
VAR MoyenneGénérale = AVERAGEX(ALL(Bureaux), [Taux_Validation])
RETURN
DIVIDE([Taux_Validation], MoyenneGénérale, 0)

// Tendance sur 30 jours
Tendance30Jours = 
VAR Actuels = [ActesTraites]
VAR Passés = CALCULATE([ActesTraites], DATEADD(Dates[Date], -30, DAY))
RETURN
DIVIDE(Actuels - Passés, Passés, 0)
